# 启动时序重构方案

## 范围

启动扫描继续按仓库并发，手动刷新、自动刷新和仓库操作保持现有顺序；改变的是所有异步结果写回前都必须拥有当前代次。代理与超时从前端请求传至实际子进程，不再写进进程级全局变量。

## 执行顺序

1. 使用 M-L4-06 异步与取消：给快照协调器增加可失效的写回租约；新任务进入时使旧租约失效。
2. 使用依赖显式化：把 Git 代理和超时封装为请求级执行参数，沿服务调用链传到 Git 进程。
3. 为命令执行器接入同一执行参数，并补充竞态、代理隔离和超时测试。

## 风险与验证

风险在于参数传递遗漏和旧任务错误提示覆盖新状态。通过 TypeScript 时序单测、Go 单测、前端构建和全量 Go 测试验证；不引入静默回退。
