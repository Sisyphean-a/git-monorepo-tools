# 变更页文件差异展开卡顿 根因分析

## 1. 问题定位

| 关键位置 | 说明 |
|---|---|
| `snapshot/file_diff.go:11` | 单文件差异请求先重建整仓基础快照，再读取目标文件差异。 |
| `snapshot/git_status.go:30` | 整仓快照额外执行状态读取和两次全仓 numstat。 |
| `src/app/components/diff-list.tsx:107` | 展开状态变化会重新执行整个文件列表的行渲染。 |
| `src/app/components/file-diff.tsx:31` | 返回后一次性把全部 diff 行创建为 DOM，没有虚拟渲染。 |
| `src/app/components/file-diff.tsx:49` | 面板卸载后没有结果缓存，旧请求只能忽略结果，不能复用。 |

## 2. 失败路径还原

**正常路径**：点击文件 → 校验仓库和文件路径 → 只读取目标文件差异 → 复用同版本结果 → 只渲染可见行。

**失败路径**：点击文件 → 重渲染全部文件行 → 后端执行整仓状态与统计 → 再启动目标文件 Git 命令 → 前端一次性创建全部行节点 → 收起后丢弃结果。

**分叉点**：`snapshot/file_diff.go:17` — 按需详情错误复用了整仓快照构建路径，使一个文件的读取承担整仓扫描成本。

## 3. 根因

**根因类型**：logic

**根因描述**：按需单文件详情没有保持独立轻量路径；后端为确认文件状态重复扫描整个仓库，前端又缺少按扫描版本隔离的结果复用和可见区域渲染，导致等待时间与仓库规模、差异行数共同增长。

**是否有多个根因**：是。主因是后端整仓重扫；次因是前端无缓存、列表无渲染隔离和 diff 全量 DOM 渲染。

## 4. 影响面

- **影响范围**：所有变更页文件差异首次展开、重复展开和快速切换。
- **潜在受害模块**：单文件差异后端接口、变更列表、差异面板；其他仓库刷新和 Git 操作不应改变。
- **数据完整性风险**：无；该路径只读。
- **严重程度复核**：维持 P1；它不损坏数据，但持续影响高频核心交互。

## 5. 修复方案

### 方案 A：保持现有交互，精简数据与渲染热路径

- **做什么**：后端直接校验仓库和相对路径后读取目标文件；前端按仓库扫描版本缓存并复用请求；文件行使用渲染隔离；diff 只渲染可见行。
- **优点**：直接消除根因，保留完整 diff 和现有交互，依赖与常驻内存增量小。
- **缺点 / 风险**：虚拟行需要保持固定行高；仓库版本变化时必须可靠失效缓存。
- **影响面**：单文件差异接口、变更页组件、对应测试和前端测试脚本。

### 方案 B：引入完整代码编辑器并改为固定分栏

- **做什么**：接入 Monaco 等编辑器，在固定面板展示结构化差异。
- **优点**：编辑器能力更完整，交互更接近 IDE。
- **缺点 / 风险**：包体、内存和启动成本明显增加，仍不能替代后端热路径优化。
- **影响面**：整体工作区布局、构建依赖、差异模型和主题系统。

### 推荐方案

**推荐方案 A**，理由：它直接处理已确认的等待与卡顿来源，不增加重型运行时，并符合按需详情和完整展示的既有架构决定。用户已确认按该方案实现。
