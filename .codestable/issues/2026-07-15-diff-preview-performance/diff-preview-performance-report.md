# 变更页文件差异展开卡顿 Issue Report

## 1. 问题现象

在变更页点击文件后，代码差异不会立即、平滑地展开；首次打开有明显等待，大型差异出现时还会发生界面卡顿。

## 2. 复现步骤

1. 打开包含变更文件的仓库。
2. 进入“变更”页。
3. 点击任意文件展开代码差异，随后切换文件或收起后再次打开。
4. 观察到：详情出现存在延迟，大型差异渲染不流畅，重复打开仍需等待。

复现频率：稳定；仓库或文件差异越大越明显。

## 3. 期望 vs 实际

**期望行为**：点击文件后应立即反馈并快速显示差异；重复打开接近即时，大型差异滚动保持流畅。

**实际行为**：首次展开需要等待，重复展开会重新读取，大型差异一次性渲染时会卡顿。

## 4. 环境信息

- 涉及模块 / 功能：变更页、单文件差异读取、差异行渲染
- 相关文件 / 函数：后端单文件差异接口、前端变更列表与差异面板
- 运行环境：Wails 桌面端，Windows
- 其他上下文：目标体验接近 VS Code 的差异查看，但保持当前工具轻量

## 5. 严重程度

**P1** — 高频核心交互持续受到影响，文件或仓库越大越明显，但不造成数据错误。
