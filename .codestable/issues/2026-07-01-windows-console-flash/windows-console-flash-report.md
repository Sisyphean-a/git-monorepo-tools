# Windows 控制台黑框闪烁 Issue Report

## 1. 问题现象

在 Windows 11 上运行本地构建出的桌面 `exe` 时，应用会弹出一堆终端黑框。除首次启动外，应用运行过程中只要触发 Git 相关信息获取，也会再次闪出一堆黑框。当前用户观察上，几乎所有 Git 相关功能都可能伴随黑框闪烁。

## 2. 复现步骤

1. 在本地构建桌面应用 `exe`。
2. 双击 `exe` 启动应用。
3. 观察到：启动过程中会连续闪出多个终端黑框。
4. 保持应用运行，并触发 Git 相关信息获取或等待后台刷新。
5. 观察到：会再次闪出多个终端黑框。

复现频率：启动阶段稳定复现；运行阶段与 Git 相关操作或后台刷新相关，但当前尚未确认具体刷新间隔。

## 3. 期望 vs 实际

**期望行为**：双击 `exe` 后应用安静启动；后续任何 Git 扫描、刷新、pull/push、状态获取都不弹控制台黑框。

**实际行为**：启动时会连续闪出多个黑框；应用运行过程中，只要触发 Git 相关操作或自动刷新，也会再闪出一堆黑框。

## 4. 环境信息

- 涉及模块 / 功能：桌面应用启动链路、Git 信息获取与后台刷新链路
- 相关文件 / 函数：待定
- 运行环境：本地构建 `exe`，Windows 11
- 其他上下文：最容易触发的具体 Git 操作暂不清楚；最近未主动修改启动链路、Git 调用或定时刷新相关代码

## 5. 严重程度

**P1** — 影响桌面应用启动体验和 Git 主路径操作体验，虽然不一定阻塞功能，但用户可见干扰强且重复出现。

## 备注

- 用户对照仓库：`E:\其他\RepoMirror`
- 用户观察：对照仓库不存在同样明显的终端黑框问题，且启动性能更好
