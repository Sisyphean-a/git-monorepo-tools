# Iteration 001

## Current Understanding

原结构的主要问题不是单纯文件多，而是入口承担业务流程、系统能力平铺在根目录、前端视图直接依赖 Wails，导致任何改造都容易穿透多层。

## Implementation Approach

连续完成后端模块化和前端分层，先建立真实接口边界，再移动实现；保留 Wails 公共方法和现有界面行为，不做视觉或产品功能改造。

## Changes This Iteration

- Go 根目录只保留应用入口；桌面能力和终端能力进入独立内部模块，Git 快照大文件按命令、状态、历史和辅助逻辑拆分。
- React 形成 domain、application、infrastructure、features、components 分层，主应用只组合控制器和界面。
- 完整应用后端端口覆盖仓库、历史、差异、AI、终端、事件和剪贴板，具体 Wails/runtime 依赖只存在于 infrastructure。
- 核心编排补充批量操作、设置刷新失败和渐进启动扫描测试；空 catch 改为显式错误或日志。
- README 记录目录职责、依赖方向和新增能力的落点。
