# 组合命令与自定义命令

## Objective

为右侧按钮区增加可区分的组合命令与自定义命令能力，支持固定操作链和自定义命令执行，并完成对应的桌面端 UI/UX 设计与实现。

## Starting Point

当前右侧操作区只有写死的暂存、生成、提交、同步按钮，设置里没有命令配置入口，也没有可承载自定义命令输出的执行面板。

## Acceptance Criteria

- 右侧操作区能区分内置按钮与用户自定义的组合/命令按钮。
- 用户可在设置中新增、编辑、删除组合命令，至少支持把现有按钮动作按顺序组合执行。
- 用户可在设置中新增、编辑、删除自定义命令按钮，并在当前仓库目录执行，例如 `wails build`。
- 执行组合命令或自定义命令后，界面能持续展示可滚动的命令输出和结果状态。
- 完成 Task agent 功能验收。

## Non-Goals

- 不重做整体视觉主题。
- 不增加云端同步或命令分享。

## Decisions And Assumptions

- 配置入口放进设置，运行入口放在右侧操作区。
- 组合命令仅编排现有内置动作；自定义命令按钮单独管理并直接执行命令行。
- 输出采用右侧下部常驻面板，不用每次弹窗打断流程。

## Current State

需求已明确，技能预检完成，正在补齐数据模型与执行链路。

## Next Action

实现设置中的命令配置模型、后端执行接口，以及右侧面板的命令区和输出区。
