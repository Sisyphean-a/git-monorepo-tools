# Iteration 001

## Current Understanding

需求核心不是单纯多几个按钮，而是把“配置入口、执行入口、输出反馈”三件事接成一条顺手的链路。

## Implementation Approach

- 设置里新增命令配置页，管理组合按钮和自定义命令按钮。
- 右侧面板保留现有内置动作，再新增“组合”“命令”区，并加常驻输出区。
- 后端新增仓库目录下执行命令的接口，前端组合动作顺序编排现有内置能力。

## Changes This Iteration

- 扩展设置模型，持久化 `commandCenter`，并给出默认组合按钮。
- 新增命令配置常量与清洗逻辑，避免旧本地设置读取失败。
- 拆分设置弹窗，把仓库、AI、Git 行为、命令配置拆成独立组件。
- 右侧操作区新增组合按钮区、自定义命令区和输出区。
- 新增仓库目录命令执行接口，并补齐 Wails 前端绑定。
- 增加前端 API 测试与 Go 命令执行测试。
