# Iteration 001

## Current Understanding

这次需求的关键不是“把终端嵌进去”这么简单，而是把仓库上下文、真实 PTY、前端渲染性能和生命周期管理一起打通，并保证切换仓库、切换 Tab、关闭应用时行为都正确。

## Implementation Approach

- 后端抽象 `terminalManager` 和 `terminalHost`，把 session 复用、输出转发、输入写入和关闭清理集中管理。
- Windows 路径显式使用 ConPTY，并用 Job Object 保证应用退出时整棵终端进程树一起回收。
- 前端增加懒加载的 `终端` Tab，用 `xterm.js` 渲染并按仓库缓存 surface，避免切仓或切 Tab 时卸载终端实例。

## Changes This Iteration

- 新增后端终端模型、session manager、Windows ConPTY host 和非 Windows PTY host。
- 在 `app.go` 与 `main.go` 接入终端 manager 初始化、Wails 事件桥接和应用关闭时的统一 `CloseAll()`。
- 新增前端 `RepoTerminalTab` 与 `RepoTerminalSurface`，在工作区顶部加入“终端”Tab，并把原头部“终端”入口改为切到该 Tab。
- 接入 `@xterm/xterm`、`@xterm/addon-fit` 与前端 API/Wails 绑定，支持交互输入、彩色输出和 resize。
- 补充 Go 测试与前端 API 测试，覆盖同仓复用、异仓隔离、cwd、Ctrl+C 和关闭事件等关键行为。
- 修复非 Windows 关闭只杀父进程的问题，改为独立进程组并按负 pgid 清理整组子进程。
