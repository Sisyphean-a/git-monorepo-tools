# Iteration 001

## 当前理解

原有文件行点击直接执行暂存或取消暂存，无法承担查看变更的职责；终端还保留一套专门把 `Shift+Enter` 转成 PowerShell 插入行的代理链路。

## 实现方案

- 删除 `Shift+Enter` 动作、代理序列、PowerShell `AddLine` 绑定和对应集成测试，保留复制、粘贴和输入队列能力。
- 新增只读文件差异请求，后端仅接受当前变更列表中的路径，并分别读取索引、工作区和未跟踪文件差异。
- 文件行改为单项展开或收起，行下方展示带基础语义着色的 unified diff；加载和错误状态显式呈现。
- 暂存操作只保留在明确的批量按钮中，文件点击不进入 Git 写操作链路。

## 本次变更

- 接通 Go 服务、Wails 绑定、前端 API 和差异展示组件。
- 对 staged、unstaged、untracked、路径越界和快捷键透传增加或调整测试。
- 删除文件级暂存切换处理和终端插入行死代码。
- 处理已跟踪文件只有新增行时也可能显示状态 `A` 的边界，改用 Git 索引事实判断是否未跟踪。
