# 变更列表内联差异

## 目标

移除终端 `Shift+Enter` 的特殊功效。变更列表中的文件点击改为只读地展开或收起真实代码差异，不再切换暂存状态。

## 起点

- Windows 终端当前会把 `Shift+Enter` 转换为 PowerShell 的插入行操作。
- 文件行当前是暂存或取消暂存按钮，点击会立即修改 Git 状态。
- 旧的独立 diff 面板和数据链路已经移除，需要按当前交互重新建立最小只读能力。

## 验收标准

- `Shift+Enter` 不再触发自定义终端行为。
- 点击文件在该行下方展开真实差异，再次点击收起；切换文件时内容正确更新。
- 已暂存、未暂存和未跟踪文件都有对应差异，加载与错误状态清楚可见。
- 文件点击不执行暂存或取消暂存；现有明确的批量 Git 操作保持正常。
- 自动化验证、独立代码审查和 Task agent 功能验收通过。

## 非目标

- 不做逐行或逐块暂存。
- 不重做整体工作区布局和视觉主题。
- 不引入第三方 diff 编辑器。

## 决策与假设

- 使用单项展开：同一时间只展开一个文件，点击同一文件可收起。
- 已暂存项读取索引差异，未暂存项读取工作区差异，未跟踪文件按新增文件展示。
- 按项目注意事项在当前检出直接实施；当前工作区起始状态干净，不创建独立 worktree。
- 本次是一个有界的交互与只读数据链路改造，直接由 goal 管理，不另建重复 feature 单元。
- 方案深度约定文件当前缺失；不采用模拟数据、静态占位或吞错，按真实 Git 输出实现。

## 当前状态
