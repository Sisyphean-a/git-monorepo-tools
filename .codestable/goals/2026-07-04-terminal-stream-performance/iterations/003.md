# Iteration 003

## Current Understanding

主链路性能优化已经完成，当前重点不再是继续挤热路径，而是把“高频输出、后台切换、恢复可见、会话复用”这些边界证据补齐，并按 CodeStable 规则处理最后的 review / acceptance gate。

## Implementation Approach

- 保持后端 `terminalOutputBatcher` 和前端逐帧 writer 的主方案不变，只补强高频隐藏态场景的资源控制。
- 把运行时事件监听抽成共享 `TerminalEventBus`，避免每个终端 surface 自己绑定一对 Wails 事件。
- 在终端隐藏时暂停 xterm 刷写，只保留有界尾部 backlog；恢复显示后再继续写入。

## Changes This Iteration

- 新增 `src/app/terminal-event-bus.ts` 与 `src/app/terminal-runtime-event-bus.ts`，把终端输出/退出事件改成共享订阅总线。
- 更新 `src/app/components/repo-terminal-surface.tsx`，终端 surface 切到隐藏态时暂停 writer，恢复时继续消费；会话订阅与退出消息也改走统一总线。
- 继续增强 `src/app/terminal-output-writer.ts`，加入 `queuedChars` 统计、隐藏态有界 backlog 裁剪和旧输出跳过提示。
- 补充 `src/app/terminal-event-bus.test.ts` 与 `src/app/terminal-output-writer.test.ts`，覆盖会话路由、暂停恢复和隐藏态海量积压裁剪路径。
- 修正了一个测试风险：原隐藏态裁剪用例没有真正越过实现里的最小批次阈值，现已改成真实大流量积压场景，验证运行时语义而不是伪边界。

## Problems Encountered
