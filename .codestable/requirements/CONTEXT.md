# 工作区上下文

- scope: workspace

## 术语

- **仓库快照**：某一时刻用于界面展示的仓库状态、文件变更和远端关系；不包含提交历史、提交详情或文件差异正文。
- **交互操作**：用户发起并需立即反馈的单仓 Git 操作，如暂存、取消暂存、提交、拉取、推送和单仓刷新。
- **后台刷新**：启动扫描、自动扫描或全局手动刷新触发的非交互式状态读取。
- **按需详情**：用户进入历史、日志或文件差异界面后才读取的数据。

## 稳定规则

- 交互操作按前台顺序执行，并以返回的单仓快照回写；后台刷新独立排队。交互开始或结束后，较早的后台结果不得覆盖交互结果。
- 提交历史、提交详情和文件差异必须按需读取，不塞进基础仓库快照。
- 所有未跟踪变更按文件展示：使用 `git status --porcelain=v1 -b --untracked-files=all`，每个未跟踪文件对应一个 `FileChange`，计数和列表使用同一粒度。
- 未跟踪文件不设静默数量上限；大仓库性能问题应优化读取或交互，不得遗漏真实变更。
- Pull 前必须显式拒绝扫描失败、有冲突或有本地变更的仓库；默认策略为 `--ff-only`，不会自动 merge、rebase 或 stash。
